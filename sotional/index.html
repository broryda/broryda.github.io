Docker Analysis

Sotional 1.0.3 웹 구조 정리

Flask + MariaDB 기반 동호회 웹앱의 실행 구조, 라우팅, 데이터 모델을 한 페이지로 요약했습니다.

1) Docker 구성

  • web: Python 3.12 + Flask 앱 (`web/Dockerfile`)
  • db: MariaDB 11.4 (`db/Dockerfile`)
  • 네트워크: `sotional-network` 브릿지 네트워크
  • 의존관계: `web`은 `db` healthcheck 통과 후 실행
  • 주요 포트: `WEB_PORT:5000`, `3306:3306`
docker compose up -d --build

2) 웹 앱 화면/기능 구조

공통

`base.html` 기반 사이드바 + 메인 컨텐츠 레이아웃

대시보드

다가오는 일정 5개 + 레이팅 TOP 5

게시판

공지 / 자유 / 리그(정기·반기·토너먼트) / 뉴스 / 건의

일정

월간 캘린더, 관리자 일정 등록/수정/삭제

레이팅

Elo 기반 점수 계산, 경기 결과 기록

계정

관리자 전용 사용자 생성/권한 관리

3) Flask 라우트 맵

  • `/` -> 로그인 여부에 따라 `/login` 또는 `/dashboard`
  • `/login`, `/logout`
  • `/dashboard`
  • `/admin/users` (관리자)
  • `/board/<board>`, `/board/<board>/new`
  • `/post/<id>`, `/post/<id>/edit`, `/post/<id>/delete`
  • `/schedule`, `/schedule/new`, `/schedule/<id>/edit`, `/schedule/<id>/delete`
  • `/rating`

4) DB 스키마 요약

  • users: 계정, 권한(admin/user), rating
  • posts: 게시판 글(보드/리그유형/이미지/작성자)
  • schedules: 일정(날짜/시간/장소/작성자)
  • rating_matches: 대국 결과 로그(승자/패자/점수변동)
